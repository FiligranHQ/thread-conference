Design System v2

Full implementation reference for the THREAD 2026 revamped landing page — color tokens, typography, components and usage rules. Built for direct hand-off to the web designer / dev team.

Changelog vs v1

✓Warm gradient (coral/pink/orange) removed everywhere. Signature gradient is now blue → cyan → lime only.
✓Button variants restricted to cool colors (blue/cyan/lime) + outline. No coral/orange buttons.
✓The 6 experience zones each get a distinct accent (top-line), cards stay outlined/dark.
✓Session cards are outlined only — no full-color fills on cards.
✓Tags/badges/speaker categories can use the full palette including coral, magenta, pink.
✓Generative mosaic art (grainy pixel blocks) extended to hero, venue map and register band.

Signature gradient

02 — Color usage rules

Cool set vs. warm set

Cool set — UI & interaction

Blue, Cyan, Lime

✓Buttons (primary + outline hover), links, focus rings, the signature gradient, primary zone accents.
✕Never used for large flat background fills.

Warm set — accent & categorization

Coral, Magenta, Pink, Purple

✓Tags, badges, card top-lines, speaker/voice category markers, zone accents (Strategy Room = purple, Simulation Zone = magenta).
✕Never used in buttons, gradients, or as a background fill behind body text.

07 — Cards & surfaces

Rules

✓All content cards (sessions, tickets, speakers, logistics) use --surface background + 1px --border, radius 14–22px.
✓Emphasis achieved via border-color change, a colored top-line, or a colored tag inside — never a solid color fill behind text.
✕Session/keynote cards are never colored solid (e.g. no more coral-filled keynote card) — outline + tag only.
08 — Mosaic / grain

Generative pixel-block art

Implementation: CSS grid of 1x1 or occasional 2x2 cells, ~45–85% fill density, colors randomized from the 6 non-neutral tokens (excluding pure white/black), each cell overlaid with an SVG fractal-noise filter at ~35–55% opacity blended with mix-blend-mode:overlay.

ContextDensityNotes
Hero background~45%Radial mask clears the center for text legibility
Venue map placeholder~60%Higher density, no mask — full block
Register band background~45%Low opacity overlay (~70%) behind CTA
Key art / social share image~85%Dense, used standalone for OG image / social

10 — Accessibility

Contrast notes

✓Blue, Cyan, Lime on --bg (#030305): pass AA for large text/UI elements and buttons (dark text on these backgrounds via --ink).
✕Coral/Magenta/Pink/Purple at full saturation fail AA for small body text on dark background — always used via the tinted-text variants shown in section 05, never raw hex on raw background for text under 16px.
✓Primary body copy stays white/muted-white only; color is never the vehicle for body text.
11 — Do & don't

Summary

✓Do keep the gradient cool-only (blue/cyan/lime).
✓Do use one distinct accent color per zone/speaker category for quick scanning.
✓Do use grain/mosaic as the recurring signature texture.
✕Don't use coral/orange/pink in any gradient or button.
✕Don't fill cards with solid brand color — outline + accent line only.
✕Don't tile the mosaic as a perfectly even grid — keep it scattered/organic with occasional clusters.
