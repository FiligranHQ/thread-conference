Design System

Full implementation reference for the THREAD 2026 landing page — color tokens, typography, components and usage rules. Built for direct hand-off to the web designer / dev team.

02 — Color usage rules

Cool set vs. warm set

Cool set — UI & interaction

Blue, Cyan, Lime, Mint

✓Buttons (primary + outline hover), links, focus rings, the signature gradient, primary zone accents.
✕Never used for large flat background fills.

Warm set — accent & categorization

Coral, Pink, Purple, Mauve

✓Tags, badges, card top-lines, speaker/voice category markers, zone accents (Strategy Room = purple, Simulation Zone = pink).
✕Never used in buttons, gradients, or as a background fill behind body text.

07 — Cards & surfaces

Rules

✓All content cards (sessions, tickets, speakers, logistics) use --surface background + 1px --border, radius 14–22px.
✓Emphasis achieved via border-color change, a colored top-line, or a colored tag inside — never a solid color fill behind text.
✕Session/keynote cards are never colored solid — outline + tag only.
08 — Mosaic / grain

Generative pixel-block art

Implementation: CSS grid of 1x1 or occasional 2x2 cells, ~45–85% fill density, colors randomized from the 8 palette tokens, each cell overlaid with an SVG fractal-noise filter. Noise treatment: Intensity 48 (overlay opacity ≈ .48) / Contrast 56 (≈ contrast(1.56) applied to the noise layer), blended with mix-blend-mode:overlay.

ContextDensityNotes
Hero background~45%Radial mask clears the center for text legibility
Venue map placeholder~60%Higher density, no mask — full block
Register band background~45%Low opacity overlay (~70%) behind CTA
Key art / social share image~85%Dense, used standalone for OG image / social

10 — Accessibility

Contrast notes

✓Blue, Cyan, Lime, Mint on --bg (#0A0A0A): pass AA for large text/UI elements and buttons (dark text on these backgrounds via --ink).
✕Coral/Pink/Purple/Mauve at full saturation fail AA for small body text on dark background — always used via the tinted-text variants shown in section 05, never raw hex on raw background for text under 16px.
✓Primary body copy stays #FBFBFB / muted #FBFBFB only; color is never the vehicle for body text.
11 — Do & don't

Summary

✓Do keep the gradient cool-only (lime/sky/blue).
✓Do use one distinct accent color per zone/speaker category for quick scanning.
✓Do use grain/mosaic as the recurring signature texture, at Intensity 48 / Contrast 56.
✓Do use only Elms Sans (titles) and IBM Plex Sans / IBM Plex Mono (UI, body, labels).
✕Don't use coral/pink/purple/mauve in any gradient or button.
✕Don't fill cards with solid brand color — outline + accent line only.
✕Don't tile the mosaic as a perfectly even grid — keep it scattered/organic with occasional clusters.
✕Don't use Inter anywhere in the system.
